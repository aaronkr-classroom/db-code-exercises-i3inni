# 5장 - 개념적 설계 기초

## 실습문제 #1

---

## 0. 요구 분석 내용

게임 회사는 여러 개의 모바일 게임을 제공하고 있다.  
회원 가입을 하려면 id와 pw를 설정해야 하고, 이름과 닉네임, 휴대폰 번호가 필요하다.  
게임 정보는 게임 번호와 게임명, 현재 버전, 최초 출시일, 현재 버전 출시일을 관리한다.  
회원이 각 게임을 언제 시작해서 언제 종료했으며, 몇 점을 획득했는지 이력을 기록한다.

---

## 1. 개체 후보 식별

| No. | 개체 후보  |
| --- | ---------- |
| 1   | 회사       |
| 2   | 게임       |
| 3   | 회원       |
| 4   | 플레이이력 |
| 5   | 버전       |
| 6   | 점수       |
| 7   | 이용       |

## 2. 개체 후보 검토

| No. | 개체 후보  | 데이터 수 | 주요 속성        | 비고           | 개체 타입 선정 여부 |
| --- | ---------- | --------- | ---------------- | -------------- | ------------------- |
| 1   | 회사       | 다수      | 회사명           | 확장 고려      | O                   |
| 2   | 게임       | 다수      | 게임번호, 게임명 | 핵심 데이터    | O                   |
| 3   | 회원       | 다수      | id, pw, 이름     | 핵심 데이터    | O                   |
| 4   | 플레이이력 | 다수      | 시작일, 종료일   | 이력 관리 핵심 | O                   |
| 5   | 버전       | 다수      | 버전정보         | 반복 데이터    | O                   |
| 6   | 점수       | 다수      | 점수             | 속성           | X                   |
| 7   | 이용       | 다수      | 없음             | 관계           | X                   |
