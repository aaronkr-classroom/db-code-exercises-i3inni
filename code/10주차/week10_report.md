## 연습문제 풀이

**1번 → ②**

"릴레이션은 매우 구체적인 개념이다"가 틀렸다. 릴레이션은 물리적 저장 구조가 아닌 추상적인 논리적 개념이다.

---

**2번 → ①**

"하나의 릴레이션은 여러 형태의 테이블로 표현될 수 있다"가 맞다. 릴레이션은 속성의 순서 개념이 없으므로 열 순서가 다른 여러 테이블로 표현될 수 있다. 반면 일반 테이블은 열 순서가 고정된다. ③·④는 반대로 설명했으므로 틀렸다.

---

**3번**

| 용어 | 설명 |
| --- | --- |
| 투플 | b. 릴레이션의 각 행(row)을 의미함 |
| 기수 | d. 하나의 릴레이션이 갖는 투플의 수 |
| 차수 | a. 하나의 릴레이션이 갖는 속성의 수 |
| 도메인 | c. 하나의 속성이 취할 수 있는 모든 값의 집합 |

---

**4번 → ④**

"릴레이션 인스턴스는 시간이 흘러도 변하지 않는 정적인 성질을 갖는다"가 틀렸다. 인스턴스는 데이터가 삽입·수정·삭제됨에 따라 변하는 동적인 성질을 가진다. 정적인 것은 스키마다.

---

**5번 → 동일한 릴레이션이다**

관계 데이터 모델에서 릴레이션은 속성(열)의 순서를 구분하지 않는다. 두 테이블은 열의 배치 순서만 다를 뿐 동일한 데이터를 갖고 있으므로 같은 릴레이션이다.

---

**6번**

후보키는 유일성과 최소성을 만족하는 속성 집합이어야 한다.

---

**7번**

각 속성 검토:

| 속성 | 후보키 여부 | 이유 |
| --- | --- | --- |
| 이름 | ✗ | 동명이인 가능 |
| 주민등록번호 | ✓ | 개인마다 유일 |
| 학번 | ✓ | 학생마다 유일 |
| id | ✓ | 시스템 내 유일 |
| 휴대폰 | ✗ | NULL 존재(이가원), 중복 가능성 있음 |
| 이수학점 | ✗ | 중복 가능 |
| 주소 | ✗ | 중복 가능 |
- 후보키: 주민등록번호, 학번, id
- 기본키: 학번 — 업무적으로 가장 일반적이고 안정적으로 식별에 사용됨

---

**8번**

- 상담사_id 단독: 유일하지 않음 (102번 상담사가 여러 건)
- 피상담자_id 단독: 유일하지 않음
- (상담사_id, 피상담자_id): 유일하지 않음 → 103번과 1302037이 두 번 등장
- (상담사_id, 피상담자_id, 상담일시): 모든 행이 유일함 ✓

기본키: (상담사_id, 피상담자_id, 상담일시)

---

**9번**

- (콘텐츠_id, 수강생_id): 유일하지 않음 → (C113, 1302037)이 두 번, (C110, 1203921)이 두 번 등장
- (콘텐츠_id, 수강생_id, 수강차시): 모든 행이 유일함 ✓

기본키: (콘텐츠_id, 수강생_id, 수강차시)

---

**10번 → 삽입 불가**

사번이 이 릴레이션의 기본키인데, 기본키는 개체 무결성 제약조건에 의해 NULL 값을 가질 수 없다. 사번이 아직 정해지지 않았다는 것은 NULL 상태를 의미하므로 삽입할 수 없다.

---

**11번 → 참조 무결성 제약조건 위반**

사원 릴레이션의 동아리번호는 동아리 릴레이션의 번호를 참조하는 외래키다. 동아리 릴레이션에 존재하는 번호는 100, 200, 300, 400, 500뿐인데, 조현수(사번 1202)의 동아리번호가 600으로 참조할 대상이 없다. 외래키 값은 반드시 참조 릴레이션에 존재해야 하므로 참조 무결성 제약조건을 위반한다. (김민수의 NULL은 외래키에서 허용되므로 위반 아님)
