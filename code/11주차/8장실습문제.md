# Chp 8 실습문제
## 대학 교수·학생·교과목 관리 데이터베이스

---

## 1. 요구사항

어느 대학에서 교수가 학생을 지도하는 관계, 학생이 교과목을 수강하는 관계, 교수가 교과목을 강의하는 관계를 관리한다.

- 교수는 **사번, 직급, 전공, 입사일** 정보를 가진다.
- 학생은 **학번, 이름, 주소, 학년** 정보를 가진다.
- 교과목은 **교과목번호, 교과목명, 학점, 구분** 정보를 가진다. (구분: 전공필수/전공선택/교양필수/교양선택)
- 교수 1명이 여러 학생을 지도하며, 지도 학생이 없는 교수도 있다.
- 모든 학생은 반드시 1명의 지도 교수가 배정된다.
- 교수가 교과목을 강의한 **년도, 학기** 정보가 필요하다.
- 동일 교과목을 여러 교수가 강의할 수 있다 (분반).
- 학생이 교과목을 수강한 **년도, 학기, 성적** 정보가 필요하다.
- 학생은 한 학기에 최대 21학점 수강 가능.
- 성적이 'F'인 경우에만 동일 교과목 재수강 가능.

---

## 2. 최종 개체 타입

| No. | 개체 타입 | 속성 목록 | 식별자 |
|-----|----------|----------|--------|
| 1 | 교수(Professor) | 사번, 직급, 전공, 입사일 | 사번 |
| 2 | 학생(Student) | 학번, 이름, 주소, 학년 | 학번 |
| 3 | 교과목(Subject) | 교과목번호, 교과목명, 학점, 구분 | 교과목번호 |

---

## 3. 최종 관계 타입

| No. | 관계 타입 | 관련 개체 | 관계 유형 | 관계 설명 속성 |
|-----|----------|----------|----------|--------------|
| 1 | 지도하다 | 교수 : 학생 | 1 : N (교수 선택, 학생 필수) | 없음 |
| 2 | 강의하다 | 교수 : 교과목 | M : N | 년도, 학기 |
| 3 | 수강하다 | 학생 : 교과목 | M : N | 년도, 학기, 성적 |

> **핵심:** 1:N 관계(지도하다)는 N쪽(학생)에 FK로 흡수, M:N 관계(강의/수강)는 별도 중간 테이블 생성

---

## 4. ER 다이어그램 (텍스트 표현)

```
[교수]──────<지도하다>──────[학생]
  사번(PK)   1       N    학번(PK)
  직급                    이름
  전공                    주소
  입사일                  학년
   │                        │
   │M                       │M
   │                        │
<강의하다>              <수강하다>
  년도, 학기             년도, 학기, 성적
   │N                       │N
   │                        │
[교과목]────────────────────┘
  교과목번호(PK)
  교과목명
  학점
  구분
```

## 5. 릴레이션 스키마 변환

```
교수( 사번, 직급, 전공, 입사일 )

학생( 학번, 이름, 주소, 학년, 사번(FK) )
  └ 지도하다 1:N → 학생 테이블에 FK로 흡수

교과목( 교과목번호, 교과목명, 학점, 구분 )

강의하다( 사번(FK), 교과목번호(FK), 년도, 학기 )
  └ 교수:교과목 M:N → 중간 테이블 생성

수강하다( 학번(FK), 교과목번호(FK), 년도, 학기, 성적 )
  └ 학생:교과목 M:N → 중간 테이블 생성
```

---

## 6. 릴레이션 스키마 단순화

**지도하다 (1:N)** → 별도 테이블 없이 학생 테이블의 `사번(FK)`으로 처리  
**강의하다, 수강하다 (M:N)** → 중간 테이블 유지 (단순화 불가)

```
교수( 사번, 직급, 전공, 입사일 )
학생( 학번, 이름, 주소, 학년, 사번(FK) )         ← 지도 관계 흡수
교과목( 교과목번호, 교과목명, 학점, 구분 )
강의하다( 사번(FK), 교과목번호(FK), 년도, 학기 )
수강하다( 학번(FK), 교과목번호(FK), 년도, 학기, 성적 )
```

---

## 7. 최초 테이블 생성 SQL

```sql
CREATE TABLE 교수 (
    사번        VARCHAR(10)  PRIMARY KEY,
    직급        VARCHAR(20)  NOT NULL,
    전공        VARCHAR(50)  NOT NULL,
    입사일      DATE         NOT NULL
);

CREATE TABLE 학생 (
    학번        VARCHAR(10)  PRIMARY KEY,
    이름        VARCHAR(20)  NOT NULL,
    주소        VARCHAR(100),
    학년        INT          NOT NULL,
    사번        VARCHAR(10),  -- 지도교수 FK
    FOREIGN KEY (사번) REFERENCES 교수(사번)
);

CREATE TABLE 교과목 (
    교과목번호  VARCHAR(10)  PRIMARY KEY,
    교과목명    VARCHAR(50)  NOT NULL,
    학점        INT          NOT NULL,
    구분        VARCHAR(20)  NOT NULL  -- 전공필수/전공선택/교양필수/교양선택
);

CREATE TABLE 강의하다 (
    사번        VARCHAR(10)  NOT NULL,
    교과목번호  VARCHAR(10)  NOT NULL,
    년도        INT          NOT NULL,
    학기        VARCHAR(10)  NOT NULL,
    PRIMARY KEY (사번, 교과목번호, 년도, 학기),
    FOREIGN KEY (사번) REFERENCES 교수(사번),
    FOREIGN KEY (교과목번호) REFERENCES 교과목(교과목번호)
);

CREATE TABLE 수강하다 (
    학번        VARCHAR(10)  NOT NULL,
    교과목번호  VARCHAR(10)  NOT NULL,
    년도        INT          NOT NULL,
    학기        VARCHAR(10)  NOT NULL,
    성적        CHAR(2),
    PRIMARY KEY (학번, 교과목번호, 년도, 학기),
    FOREIGN KEY (학번) REFERENCES 학생(학번),
    FOREIGN KEY (교과목번호) REFERENCES 교과목(교과목번호)
);
```

## 8. 데이터 입력 예시

```sql
INSERT INTO 교수 VALUES
('P001', '교수', '컴퓨터공학', '2010-03-01'),
('P002', '부교수', '데이터베이스', '2015-09-01'),
('P003', '조교수', '인공지능', '2020-03-01');

INSERT INTO 학생 VALUES
('S001', '김철수', '서울시 강남구', 3, 'P001'),
('S002', '이영희', '서울시 마포구', 2, 'P001'),
('S003', '박민준', '경기도 수원시', 4, 'P002');

INSERT INTO 교과목 VALUES
('CS101', '데이터베이스', 3, '전공필수'),
('CS102', '자료구조', 3, '전공필수'),
('CS103', '인공지능개론', 3, '전공선택');

INSERT INTO 강의하다 VALUES
('P001', 'CS101', 2026, '1학기'),
('P002', 'CS101', 2026, '1학기'),
('P003', 'CS103', 2026, '1학기');

INSERT INTO 수강하다 VALUES
('S001', 'CS101', 2026, '1학기', 'A'),
('S001', 'CS102', 2026, '1학기', 'B+'),
('S002', 'CS101', 2026, '1학기', 'F'),
('S002', 'CS101', 2025, '2학기', 'F');  -- F여서 재수강 가능
```
