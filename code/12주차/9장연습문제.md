## 연습문제

**1.** 개체 통합이 가능하지 않은 것은?
- 정답: 모든 속성의 도메인이 동일한 개체 타입들

**2.** 이력 데이터에 대한 설명이 잘못된 것은?
- 정답: 변경 이력은 데이터가 변경될 때마다 변경 후의 상태를 확인해야 할 때 발생한다.

**3.** 개체 무결성 제약조건이 만족해야 하는 세 가지 규칙에 해당하지 않는 것은?
- 정답: 기본키는 데이터 보안이 요구되어야 한다.

**4.** 참조 무결성에 대한 설명이 잘못된 것은?
- 정답: 외래키는 널 값을 가질 수 없다.

**5.** 참조 무결성의 세부 업무 규칙에 대한 설명이 잘못된 것은?
- 정답: 제한(Restrict): 대응하는 자식 인스턴스가 없는 경우 부모 인스턴스의 삭제/수정이 불가능하다.

**6.** 명시적으로 정의해야 하는 도메인 무결성에 해당하지 않는 것은?
- 정답: UNIQUE 제약조건

**7.** 데이터 표준화의 일반 원칙이 잘못 설명된 것은?
- 정답: 서술적 정의만으로 의미 전달이 가능하도록 상세히 작성한다.

**8.** 데이터 표준의 주요 대상에 해당하지 않는 것은?
- 정답: 표준 개체

**9.** 데이터 표준화 절차가 바르게 작성된 것은?
- 정답: 요구사항 수집 → 데이터 표준 정의 → 데이터 표준 확정 → 데이터 표준 관리

```
고객구분코드(고객코드, 실적기준, 할인율, 제공마일리지)
```
- 기본키: 고객코드

**11.** 클럽 회원의 행사 참석 내역 ER → 릴레이션 스키마

```
회원(회원번호, 이름, 연락처)
행사(행사번호, 행사명, 행사일자, 장소)
참석(회원번호, 행사번호, 참석일시)
```
- 회원 PK: 회원번호
- 행사 PK: 행사번호
- 참석 PK: (회원번호, 행사번호)
  - FK: 회원번호 → 회원, 행사번호 → 행사

**12.** 아파트 재건축 공사 진행 상황 관리

```
아파트(건물번호, 이름, 주소, 시공사, 세대수, 입주시작일)
진행단계(건물번호, 순서, 주관사, 연락처, 시작일자, 종료일자)
```
- 아파트 PK: 건물번호
- 진행단계 PK: (건물번호, 순서)
  - FK: 건물번호 → 아파트(건물번호)

**13.** 주식 주가 변동 이력 관리

```
주식(종목코드, 종목명, 업종, 총주식수)
주가이력(종목코드, 변동일시, 주가)
```
- 주식 PK: 종목코드
- 주가이력 PK: (종목코드, 변동일시)
  - FK: 종목코드 → 주식(종목코드)
