# 연습문제

1. ER 모델을 구성하는 세 가지 기본 요소가 아닌 것은?
   Answer: 2번 (정보)

⸻

2. 개체 타입과 개체 인스턴스의 차이점을 설명하시오.
   Answer:
   개체 타입: 같은 속성을 가진 개체들의 집합 (예: Student)
   개체 인스턴스: 실제 저장된 하나의 데이터 (예: 특정 학생 한 명)

⸻

3. 다음은 쇼핑몰 DB의 회원 개체에 필요한 속성들이다. 각 속성의 종류가 잘못 연결된 것은? 답에 대한 이유도 설명하시오.
   Answer: 2번

이유:
휴대폰번호는 값이 변할 수 있으므로 상수 속성이 아니다.

⸻

4. 어떤 건물을 어떤 건축가가 건축했는지를 표현하는 ER 모델의 구성 요소 가운데 성격이 다른 것은? 답에 대한 이유도 설명하시오.
   Answer: 3번

이유:
건축은 관계이고 나머지는 속성이다.

⸻

5. 다음은 동아리 DB에서 회원 개체 타입과 행사 개체 타입 간의 참여하다 관계를 표현한 것이다. 적합한 관계 유형을 [________] 안에 쓰시오
   Answer: M:N

⸻

6. 다음은 경매 사이트 DB에서 작품 개체 타입과 입찰자 개체 타입 간의 낙찰되다 관계를 표현한 것이다. 적합한 관계 유형을 [________] 안에 쓰시오
   Answer: 1:1

⸻

7. 어떤 컴퓨터가 어떤 회사에서 생산되었는지를 표현하는 ER 모델에서 다음 각각은 어떤 구성 요소에 해당하는지 답하시오.

Answer: 1. 컴퓨터 → 개체 2. 생산 → 관계 3. 모델명 → 속성 4. 생산년도 → 속성 5. 대표자 → 속성 6. 메모리사이즈 → 속성 7. 회사 → 개체 8. 주소 → 속성

⸻

8. 은행에서 각 대출에 대한 상환 내역을 관리하려고 한다.
   Answer:

개체: 대출, 대출상환
관계: 상환하다
카디널리티: 1:N
특징: 대출상환은 약한 개체 (대출 삭제 시 함께 삭제)

⸻

9. 교수 1명이 최대 20명의 학생을 지도하고…
   Answer:

관계: 지도하다
카디널리티: 교수 (020) : 학생 (01)

⸻

10. 대학입시 DB에서 수험생 개체 타입과 학과 개체 타입 간의 관계
    Answer:

수험생 → 전체 참여
학과 → 부분 참여
