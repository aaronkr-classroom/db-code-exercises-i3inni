### 연습문제

1. 요구 수집 단계의 주요 업무가 아닌 것은? [ **1** ]

2. 요구 수집 명세서를 작성할 때 포함해야 할 항목에 해당하지 않는 것은? [ **2** ]

3. 요구 분석 단계의 주요 업무가 아닌 것은? [ **4** ]

4. 요구 분석이 어려운 이유가 무엇인지 설명하시오.
   **정답: 사용자의 요구사항이 모호하고 계속 변화하며, 사용자 간 요구사항이 상충되거나 개발자와의 소통 차이가 발생하기 때문입니다.**

5. 요구 수집을 위한 인터뷰 시 유의 사항으로 옳지 않은 것은? [ **4** ]

6. DB 구축의 전체 단계에 걸쳐서 요구 분석 단계가 가장 중요한 이유가 무엇인지 설명하시오.
   **정답: 설계의 기초가 되는 단계로, 여기서 오류가 발생하면 이후 단계의 수정 비용이 막대하게 발생하기 때문입니다.**

7. 다음 중 요구사항의 특정에 해당하지 않는 것은? [ **2** ]

8. 요구사항분석 시 고려 사항으로 옳지 않은 것은? [ **1** ]

9. 다음 중 데이터 요구 분석 명세서를 작성할 때의 유의 사항으로 옳은 것은? [ **2** ]

10. ( ) 안에 적합한 용어는?
    **정답: 요구 분석 명세서**

11. 다음 중 트랜잭션 요구 분석 명세서를 작성할 때의 유의 사항으로 잘못된 것은? [ **1** ]

12. 요구 분석 명세서를 작성하면서 ‘회원의 출생지’를 표현할 때 추상화 수준이 가장 적절한 것은? [ **3** ]
